Acerca del Proyecto

¿Qué buscamos lograr?

Diseñar un sistema de IA que identifique hogares en condición de pobreza estructural sin depender del ingreso monetario directo.

🎯

Predicción Precisa

Modelo XGBoost con AUC-ROC de 0.82, capaz de clasificar hogares como pobres o no pobres utilizando variables estructurales: hacinamiento, educación del jefe de hogar, equipamiento y material de vivienda.

🔍

Explicabilidad (XAI)

Implementación de SHAP (SHapley Additive exPlanations) para explicar cada predicción. El modelo no actúa como "caja negra" — cada resultado es interpretable y auditable por tomadores de decisión.

📊

Datos Oficiales del INE

Entrenado con los microdatos oficiales de la Encuesta de Hogares 2023 (BOL-INE-EH-2023) del Instituto Nacional de Estadística. Tres módulos fusionados: Vivienda, Equipamiento y Persona.

⚙️

Arquitectura MLOps

Pipeline automatizado con CI/CD: GitHub Actions, Docker, MLflow y FastAPI. El modelo se reentrena automáticamente cuando el INE publique nuevos datos de la Encuesta de Hogares.

🏥

Impacto Social

Las predicciones permiten al Estado optimizar la asignación de la Renta Dignidad y otros programas sociales, eliminando errores de inclusión/exclusión en la focalización de beneficiarios.

🌐

API REST en Tiempo Real

Endpoint de predicción con FastAPI que recibe los datos del hogar y retorna la clasificación con probabilidad, nivel de confianza e interpretación — listo para integración gubernamental.

Metodología

Pipeline CRISP-DM

El proyecto sigue la metodología estándar de minería de datos aplicada en 6 fases.

Fase 1 · Comprensión del Negocio

Análisis de la problemática de pobreza multidimensional en Bolivia. Definición de la pobreza estructural como variable objetivo binaria.

Fase 2 · Comprensión de Datos

Exploración de los microdatos de la EH-2023: 12,815 hogares, 40,647 personas, 217,855 registros de equipamiento. Identificación de 11 features clave.

Fase 3 · Preparación de Datos

Fusión de 3 datasets por folio, construcción de índice de equipamiento, hacinamiento, codificación One-Hot y balanceo SMOTE.

Fase 4 · Modelado

XGBoost con 300 árboles, max_depth=6, learning_rate=0.05. Validación cruzada estratificada 5-Fold con AUC-ROC = 0.8182 ± 0.0052.

Fase 5 · Evaluación

AUC-ROC en test: 0.7598. Explicabilidad global con SHAP beeswarm, importancia de features y análisis de predicciones individuales con waterfall.

Fase 6 · Despliegue

API REST con FastAPI, frontend web interactivo, serialización del modelo (.pkl), y arquitectura MLOps con CI/CD automatizado.

Variables

Features del Modelo XGBoost

Las 11 variables predictoras seleccionadas de los microdatos de la Encuesta de Hogares 2023.

# Variable Tipo Descripción
1 hacinamiento Numérica Ratio miembros del hogar / dormitorios
2 anios_educ_jefe Numérica Años de escolaridad del jefe/a del hogar
3 indice_equipamiento Numérica Proporción de 17 bienes duraderos que posee el hogar [0–1]
4 afiliacion_afp Binaria 1 si el jefe aporta actualmente a la Gestora/AFP
5–10 material_vivienda_* Dummy 6 dummies del material de paredes (base: adobe/tapial)
11 area_urbana Binaria 1 si el hogar es urbano, 0 si es rural
